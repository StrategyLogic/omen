# 向量数据库战场：Vector Database vs AI Memory

这是 Omen 的第二个相邻案例，用于验证同一套 `simulate / explain / compare` 工作流能否复用于不同案例。

## Case Summary

本案例关注：当 AI Memory 平台不断吸收检索、记忆、上下文编排能力时，向量数据库的核心价值会如何演化。

我们关心的不只是“谁替代谁”，还包括：

- 哪些能力会被 AI Memory 先吸收
- 哪些能力仍由向量数据库主导
- 哪些策略会把结果从“替代”拉回“融合”

## Strategic Actors

- **vector-db-incumbent**：向量数据库主导方，优势是检索性能与成熟生态
- **ai-memory-platform**：AI Memory 平台方，优势是语义编排与 Agent 工作流协同
- **app-builder**：应用建设方，决定预算、迁移节奏与架构取舍

## Capability Focus

- semantic_retrieval
- context_memory
- operational_consistency
- developer_velocity

## Expected Omen Outputs

- 结果分类（replacement / convergence / coexistence）
- 关键分叉点与因果链
- 在语义条件变化下的反事实差异
